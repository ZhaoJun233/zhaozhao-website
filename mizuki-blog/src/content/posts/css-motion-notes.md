动效很容易成为页面上声音最大的人。按钮漂浮、卡片弹跳、背景不断缩放，单独看都很精致，叠在一起却会抢走正文的注意力。我为这个演示博客制定的原则是：动画只解释状态、提示方向或营造极轻的环境感；如果移除后信息仍完整，就必须允许它安静退场。文中的持续时间与位移是**界面演示参数**，不是适用于所有产品的固定标准。

## 先问它为何移动

每加一个动画，我会先写下目的，再决定属性：

- 导航抽屉需要说明“从哪里出现”，因此使用短距离位移和透明度；
- 按钮悬停只确认可交互，颜色变化比放大更稳定；
- 花瓣属于装饰，频率要低，也不能遮住链接或正文；
- 阅读进度反映真实状态，不额外添加回弹效果。

若一句话说不清目的，这段动画多半可以删除。尤其避免让大面积图片持续缩放，因为它既干扰阅读，也可能放大裁切问题。

## 小幅度与统一节奏

界面反馈通常控制在一百五十到二百四十毫秒，位移保持在数个像素。不同组件共用少量时间变量，页面就不会像由几套节拍拼在一起。下面是本站按钮反馈的简化示例：

```css
.action {
  transition: color 180ms ease, transform 180ms ease;
}

.action:hover {
  transform: translateY(-2px);
}

@media (prefers-reduced-motion: reduce) {
  .action { transition: none; }
  .action:hover { transform: none; }
}
```

## 把“减弱动态”当作完整状态

`prefers-reduced-motion` 不只是把时长调短。持续飘动、自动打字与平滑滚动应直接停止，状态变化则保留即时反馈。更多行为差异可查阅 [MDN 的减少动态媒体特性说明](https://developer.mozilla.org/zh-CN/docs/Web/CSS/@media/prefers-reduced-motion)。实现后还要用系统设置实际切换一次，确认内容不会依赖动画结束才出现。

## 最后的检查

我会依次用键盘、触屏和减弱动态模式浏览页面，观察焦点是否清楚、元素是否位移到邻近文字上方、动画是否导致布局抖动。克制并不等于完全静止，而是让移动有来由、有边界，也知道何时停下。当读者只感到界面自然，却不必意识到每一段过渡时，动效才真正完成了工作。
