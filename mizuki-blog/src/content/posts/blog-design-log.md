我把个人博客想成一间可以反复整理的网络小屋：门口要容易找到，书架要能继续增加，窗外的风景很好看，但不能挡住阅读。233昭 的文章和项目包含可替换的**演示资料**；这篇记录描述的是样例站点的设计过程，不暗示未经说明的私人经历。

## 从内容而不是卡片开始

最初草图塞满了圆角卡片：简介一张、标签一张、时间线又一张。信息看似整齐，页面却像许多互不相干的小盒子。后来我只为重复出现的文章与项目保留卡片，其余部分改成全宽区段，用留白、细线和标题层级建立节奏。正文宽度限制在适合中文阅读的范围，长标题允许自然换行，不通过缩小字号勉强塞进一行。

视觉上以白色与炭黑为基础，海青和樱粉只负责强调。这样既保留轻盈的动画气质，也避免整个界面被单一色相淹没。圆角、阴影和边框各自只有少量规格，新增组件时先复用，而不是继续发明例外。

## 同一幅图，两种构图

桌面首屏利用插画左侧留白叠放标题，前景图保持完整比例；移动端则把图像和文字拆成上下两段，避免文字压住人物。关键规则可以浓缩为：

```css
.hero-art { object-fit: contain; }

@media (max-width: 899px) {
  .hero { grid-template-columns: 1fr; }
  .hero-copy { order: 2; }
}
```

为减少布局跳动，图片容器预先声明宽高比，卡片网格也使用稳定的列定义。关于不同尺寸资源的选择，我参考了 [web.dev 的响应式图片指南](https://web.dev/articles/responsive-images)，再根据这幅插画的实际焦点做取舍。

## 深色不是简单反相

深色主题分别调整正文、边界和图片遮罩，而不是给整个页面套滤镜。海青在暗背景上降低饱和度，樱粉则提高明度，确保链接与焦点环仍有清楚对比。主题脚本在页面绘制前读取偏好，减少载入时的明暗闪烁。

## 留下可继续修改的边界

最后，我用三条问题检查每个决定：

- 它是否直接帮助阅读，而不是只在截图里显得热闹；
- 它是否能在手机、键盘操作和减弱动态模式中成立；
- 它是否允许未来替换内容，而不必重做整个页面。

网络小屋不需要一次装修完毕。只要结构稳定、素材来源清楚、装饰知道退后，新的文章来到时就总能找到合适的位置。
