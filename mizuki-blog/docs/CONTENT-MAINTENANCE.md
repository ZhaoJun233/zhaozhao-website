# 内容维护说明

博客采用“内容与模板分离”的方式维护。博主日常只需要访问
`http://127.0.0.1:4321/admin/`，无需修改 Astro 组件。

## 后台内容入口

- **文章**：新建、修改、下线文章，设置分类、标签、封面和精选状态。
- **项目**：维护项目正文、状态、链接和精选状态。
- **站点设置**：修改昵称、站点标题、网站描述、个人简介和头像。
- **页面内容**：修改导航、页脚、首页、关于、友链、留言、鸣谢和索引页文案。
- **视觉素材**：替换首页主视觉和关于页插画，并维护素材来源。
- **分类管理**：新增、重命名、排序文章分类及其说明。

## 发布机制

后台保存后，CMS 将内容写入 `src/data/` 或 `src/content/`。Astro SSR
服务会在每次请求时读取并校验最新内容，刷新前台即可看到变化，不需要重新构建整站。
文章、项目、分类、友链、页面文案和搜索结果都使用同一个运行时内容仓库。

## 数据约束

`src/data/content.ts` 会在构建阶段校验页面内容：

- 所有必填文案去除首尾空格后必须有内容。
- 站内导航必须以单个 `/` 开头且链接不能重复。
- 外部链接必须使用 `http://` 或 `https://`。
- 时间线月份使用 `YYYY-MM` 格式。
- 视觉图片必须保存在 `src/assets/backgrounds/`。
- 列表设置了合理的最大数量，避免误操作拖垮页面。

## 长期维护边界

后台负责读者可见的内容、顺序和素材。页面结构、响应式布局、无障碍标签、搜索逻辑、发布脚本与样式由代码维护。升级 Astro 或 Decap CMS 前，先运行 `npm run check`、`npm test` 和 `npm run build`。
