# “此刻”音乐、天气与时间页面设计

## 目标

为 233昭个人博客新增 `/now/` 页面，将访客所在区域的天气、本地时间与博主后台维护的网易云歌曲整合成“海边窗景”主题页面。页面需要延续现有二次元博客的柔和海边视觉，同时保持 Cloudflare Workers、D1 与移动端架构的长期可维护性。

## 网易云能力边界

用户提供的网易云开放平台文档介绍的是网易云音乐 CLI。官方说明个人应用目前主要使用 `ncm-cli`，需要登录、App ID、Private Key 和本地播放器后端；CLI 方案不适合直接运行在 Cloudflare Worker 或访客浏览器内。

本功能采用网页混合接入：本站数据库维护歌曲展示信息和网易云歌曲 ID，前端按歌曲 ID 加载网易云嵌入播放器，并提供网易云歌曲页面链接。App ID、Private Key 和用户登录态不进入网页端。

## 页面信息架构

新增主导航项“此刻”，指向 `/now/`。页面按以下层级组织：

1. 时间与访客所在区域概览；
2. 当前天气和天气寄语；
3. 当前选曲与网易云嵌入播放器；
4. 后台启用的歌曲列表；
5. 定位、天气或音乐不可用时的局部状态。

桌面端使用杂志式双栏构图：时间窗景占据主区域，天气卡位于右侧，播放器与歌曲列表横向展开。手机端改为“时间、天气、播放器、歌曲列表”的单列顺序，所有主要操作保持在拇指可触达范围内。

## 视觉方向

采用已确认的 C 方案“海边窗景”：

- 背景使用低饱和天空蓝、晚霞粉和沙滩暖色渐变；
- 时间使用大号衬线字体，作为页面第一视觉焦点；
- 天气卡使用半透明深色玻璃效果；
- 播放区使用黑胶唱片意象，但实际控制保留网易云嵌入播放器；
- 页面根据访客本地时间切换清晨、白天、黄昏和夜晚色调；
- 黑胶及天气装饰动画遵循 `prefers-reduced-motion`；
- 使用现有设计令牌、页头和响应式断点，不另建冲突的全局主题。

## 音乐数据模型

新增 D1 表 `music_tracks`：

- `id`：内部稳定标识；
- `title`：歌曲名称；
- `artist`：歌手名称；
- `netease_song_id`：网易云歌曲 ID；
- `cover`：可选的本站媒体地址；
- `note`：可选的推荐语；
- `enabled`：是否前台展示；
- `sort_order`：展示顺序；
- `created_at`、`updated_at`：维护时间。

网易云歌曲 ID 必须为纯数字。后台拒绝重复 ID。删除歌曲记录时，若封面属于该歌曲且未被其他内容引用，则沿用现有媒体清理队列删除相关文件。

页面标题、引导文字、天气寄语模板和空状态文案保存到现有 `site_settings`，并纳入后台页面内容管理及数据导入导出。

## 后台音乐管理

新增 `/admin/music/`，沿用现有单管理员会话和后台布局，提供：

- 新建、编辑和删除歌曲；
- 上传、预览、更换和移除封面；
- 启用或隐藏歌曲；
- 拖动或按钮调整顺序；
- 根据网易云歌曲 ID 预览嵌入地址和外部歌曲链接；
- 清晰显示歌曲 ID 重复、格式错误和保存失败原因。

后台不保存网易云账号密码、登录 Cookie、App ID 或 Private Key。

## 播放交互

页面初次打开时不自动播放。访客点击歌曲后：

1. 当前歌曲区域切换标题、歌手、封面和推荐语；
2. 只创建一个网易云嵌入播放器，并替换为选中歌曲；
3. 黑胶进入播放视觉状态；
4. 页面提供“在网易云音乐中打开”链接。

由于跨域嵌入限制，自定义页面不伪造或代理网易云播放进度。真实播放、暂停和进度控制由网易云嵌入播放器负责；本站黑胶动画只表达当前选中状态。播放器加载失败时保留歌曲信息和网易云外链。

## 天气与时间数据流

新增 `/api/weather`：

1. 浏览器首次进入页面时请求定位权限；
2. 获得授权后将经纬度发送给本站天气接口；
3. 未授权或定位失败时，接口使用 Cloudflare 请求中的大致经纬度；
4. 接口向 Open-Meteo 请求当前天气；
5. 返回本站统一的精简天气结构。

接口验证经纬度范围，仅接受有限精度数值，不记录访客坐标。天气结果按坐标网格缓存约 10 分钟，避免每次访问直接请求外部服务。

区域名称使用 Cloudflare 提供的大致城市信息；该信息缺失时显示“访客所在区域”，不调用额外的反向地理编码服务。返回结构包含：区域显示名、天气代码、温度、体感温度、湿度、风向、风速和观测时间。页面根据天气代码映射本站图标与中文描述。

时间、日期、星期和问候语由浏览器 `Intl.DateTimeFormat` 按访客本地时区更新。页面在标签页重新可见时校正时间，不使用高频后台计时。

## 降级与错误处理

- 浏览器拒绝定位：静默切换到 Cloudflare 大致位置，不反复申请权限；
- Cloudflare 位置缺失：使用通用位置并显示“访客所在区域”；
- Open-Meteo 超时或异常：时间和音乐继续工作，天气区域显示“天气暂时藏进云里了”；
- 网易云嵌入失败：保留歌曲信息及外部歌曲链接；
- 浏览器阻止自动播放：等待访客点击，不显示误导性错误；
- 歌曲列表为空：显示后台可维护的设计化空状态；
- JavaScript 不可用：服务端仍输出时间页面结构、歌曲列表和网易云外链。

## 安全与隐私

- 不保存访客经纬度、城市或天气查询历史；
- `/api/weather` 只代理固定的 Open-Meteo 主机，不接受任意上游 URL；
- 对经纬度、歌曲 ID、标题和外链进行服务端验证；
- 网易云外链使用安全的 `rel` 属性；
- 不向客户端发送管理员凭证或第三方私钥；
- 天气接口设置合理超时、缓存和错误响应，避免外部服务拖慢页面。

## 测试与完成标准

自动化测试覆盖：

- 音乐歌曲的 D1 CRUD、重复 ID、启用状态和排序；
- 后台歌曲封面上传、替换与删除清理；
- `/api/weather` 的坐标验证、Cloudflare 位置降级、缓存和上游错误；
- 时间问候与昼夜主题映射；
- 网易云歌曲选择、单一嵌入实例和外链降级；
- 桌面与 390px 手机布局；
- 定位允许、拒绝和失败状态；
- `prefers-reduced-motion`、键盘操作、可访问名称及无横向溢出；
- 现有文章、留言、后台和 Cloudflare 部署回归测试。

完成条件：管理员能从后台长期维护歌曲，访客能在桌面和手机端看到本地时间及所在区域天气、选择歌曲并通过网易云播放器收听；任何一个外部服务异常时，其余页面功能仍保持可用。
