# 导航天气与全站选曲设计

## 目标

在不删除或重排首页原有内容的前提下，利用桌面导航品牌右侧的空白区域显示实时天气，并让导航音乐播放器在任意页面都能选择和控制后台已启用的歌曲。

## 导航天气

- 在“233昭”品牌右侧显示一行无边框、低干扰的信息：天气符号、地区和整数温度。
- 仅在桌面端显示；导航空间不足以及移动端隐藏，避免挤压主导航和操作按钮。
- 复用现有 `/api/weather/` 接口、天气代码映射和访客 IP 定位，不增加新接口或后台配置。
- 首次进入页面获取天气，成功后定时刷新；点击天气信息可手动刷新。
- 请求期间保留已有结果；首次请求失败时显示“天气暂不可用”，不会影响导航操作。
- 支持明暗主题和首页透明覆盖导航状态，保持文字可读。

## 导航选曲

- 复用 `Header` 已取得的后台启用歌曲列表，在导航播放器浮层中显示紧凑曲目列表。
- 浮层提供关键词输入框，在浏览器内即时按歌曲名、歌手和推荐语筛选，不新增搜索接口。
- 点击曲目后优先把后台保存的 `audioUrl` 载入全局原生 `<audio>`；留空时使用歌曲 ID 请求 `/api/music/audio/[id]/`，由 Cloudflare 解析网易云临时音频并重定向到 HTTPS CDN，同时更新标题、歌手、网易云外链及当前选中状态。
- 使用 `transition:persist="site-music-player"` 持久化唯一的全局 `<audio>`，保持站内切页不中断播放。
- 首页和导航栏保留各自独立的播放/暂停、进度、音量 UI，通过 `site:music-command`、`site:music-state` 和 `site:music-state-request` 控制并订阅同一个音频实例。
- 优先使用后台保存的封面；缺少封面时通过只读 `/api/music/covers/` 按启用歌曲自动解析并缓存网易云专辑封面。选曲事件同步 `coverUrl`，首页唱片显示该封面并缓慢旋转；解析失败时保留默认唱片。
- 没有启用歌曲时显示空状态；网易云因版权或上游异常无法提供音频时不播放，但仍保留网易云外链并允许选择其他歌曲。
- 搜索无结果时显示明确空状态；清空关键词后恢复后台排序的完整歌曲列表。
- 桌面端使用现有浮层，移动端继续使用固定浮层并保证歌曲列表可滚动、触控目标足够大。

## 边界

- 不删除或重排首页原有内容。
- 不新增天气或音乐独立页面。
- 后台音乐管理保留可选“音频播放地址”作为覆盖音源；留空时标记为“网易云在线”，启用状态、排序和最终播放地址同步到首页与导航栏。
- 不改变第三方音乐版权和可播放性限制。

## 验证

- 组件或单元测试覆盖天气显示格式、音乐选择数据和关键词匹配。
- 端到端检查桌面导航天气、移动端隐藏、任意页面选曲、首页同步与切页持久播放。
- 运行项目类型检查和相关测试。
