# Hero 天气音乐浮层设计

## 目标

将首页天气与音乐从 Hero 下方的独立大区块迁入首页首屏图片内部，形成可展开、可隐藏的半透明玻璃浮层。首页原有标题、简介、打字文案、按钮、插画、站点近况及后续内容保持不变。

同时修复精确定位时地名与坐标不一致的问题，并让天气在浮层展开期间按天气上游的更新粒度持续刷新。

## 布局与视觉

桌面端浮层位于 Hero 右下区域：右侧距离使用 `max(1.5rem, calc((100vw - var(--content-width)) / 2))`，底部距离为 4.5rem，避开左侧标题、主要按钮、人物脸部和“向下阅读”入口。浮层宽度使用 `clamp(20rem, 30vw, 27rem)`，最大高度为 Hero 可视高度的 58%，超出内容在浮层内部滚动。

浮层使用浅色半透明玻璃效果：

- 暖白色背景 `rgb(248 248 244 / 62%)`；
- 1 像素低对比边框；
- 20 像素背景模糊；
- 轻量阴影和细微高光，不使用新的大面积渐变背景；
- 保留 Hero 插画作为浮层后的主要视觉；
- 深色主题使用 `rgb(18 27 31 / 72%)` 背景，并保持 WCAG AA 文字对比度。

天气区域改为紧凑摘要：地名、天气符号、温度和天气状态为第一行，体感、湿度和风速为第二行，天气寄语位于下方。音乐区域显示当前歌曲、网易云入口和可滚动歌曲列表，初次载入仍不创建 iframe。

桌面端浮层默认展开。手机端默认收起，只在 Hero 图片右下显示“天气 · 音乐”按钮；展开后以距图片左右各 1rem、底部 1rem 的底部抽屉覆盖 Hero 图片下部，最大高度为 70svh，并可内部滚动。浮层收起时不改变 Hero 图片、文案和后续内容的布局高度。

## 组件边界

`HomeHero.astro` 继续负责 Hero 图片、文案、装饰和浮层定位，只新增一个命名插槽用于接收天气音乐内容，不读取音乐数据，也不负责天气请求。

首页编排改为：

```astro
<HomeHero>
  <HomeWeatherMusic slot="weather-music" tracks={tracks} content={editorial.nowPage} />
</HomeHero>
```

`HomeWeatherMusic.astro` 继续负责天气、音乐、浮层标题、隐藏按钮和局部样式。首页中站点近况后不再单独渲染天气音乐区块。

`WeatherPanel.astro` 和 `MusicPlayer.astro` 保持数据与语义组件边界，但使用 Hero 浮层的紧凑样式。天气 API、音乐 API、D1 表、封面和备份结构不改变。

## 展开与隐藏交互

浮层根节点提供稳定标识 `#weather-music`。展开按钮使用：

- `aria-controls="hero-weather-music-panel"`；
- `aria-expanded="true|false"`；
- 展开时文案“隐藏天气音乐”；
- 收起时文案“天气 · 音乐”；
- 至少 44×44 像素的触控区域；
- 键盘可操作，并保留明显焦点样式。

首次访问时，桌面端默认展开，手机端默认收起。用户操作后将状态保存到 `localStorage` 的 `hero-weather-music-open`，后续访问优先恢复用户选择，而不是重新使用屏幕默认值。

无 JavaScript 时浮层保持展开，天气占位和音乐外链仍可访问。JavaScript 启用后才应用响应式默认状态和用户保存状态。

收起浮层只隐藏界面，不销毁已经创建的网易云 iframe；已开始的播放可继续，用户可重新展开浮层操作播放器。隐藏时所有面板内部控件从键盘焦点顺序移除，重新展开后恢复。

## 精确定位与地名

当前接口在收到浏览器精确坐标后仍使用 Cloudflare IP 城市名，导致天气坐标正确但地名错误。新设计按坐标来源区分地名：

1. 浏览器提供精确坐标时，Worker 使用相同坐标请求固定的反向地理编码上游；
2. 反向地理编码固定为 `https://api.bigdatacloud.net/data/reverse-geocode-client`；
3. 请求只允许 Worker 内部构造，不接受访客提供任意上游 URL；
4. 请求参数为纬度、经度和 `localityLanguage=zh`；
5. 返回地名优先组合“区县 · 城市”，缺少区县时显示城市，缺少城市时显示省级区域；
6. 反向地理编码失败时显示“当前位置”，不得回退到不匹配的 Cloudflare IP 城市名。

浏览器未授权定位或定位失败时，继续使用 Cloudflare 坐标和城市名。Cloudflare 信息也缺失时使用既有天气兜底坐标，并显示“访客所在区域”。

反向地理编码将经纬度分别除以 0.02 后四舍五入生成缓存网格，缓存 60 分钟。缓存只存在于 Cloudflare Cache，不写入 D1、KV 或访问历史。

## 天气实时更新

天气上游及本站天气缓存保持 10 分钟粒度，因此“实时更新”定义为：

- 浮层首次展开时立即获取当前位置和天气；
- 浮层保持展开且页面可见时，每 10 分钟重新获取一次坐标和天气；
- 页面从后台恢复可见时，如果距离上次成功更新已超过 10 分钟，则立即刷新；
- 用户收起后停止定时器和新的定位请求；
- 用户重新展开时立即重新定位并刷新；
- 每次刷新使用 `getCurrentPosition`，不使用持续 `watchPosition`，避免长期占用定位和增加电量消耗。

界面显示最近观测时间。请求进行中保留上一份成功天气，不把面板重置为空状态；失败时显示局部提示，并在下一个刷新周期重试。

## 隐私与安全

- 经纬度只用于当次天气和反向地理编码请求；
- 不将坐标、地名或查询历史写入 D1、KV、日志结构或浏览器持久存储；
- `localStorage` 只保存浮层展开状态，不保存位置；
- 天气和反向地理编码均限制为固定上游主机；
- 坐标继续执行范围验证；
- 外部请求继续使用超时、缓存和统一错误响应；
- 反向地理编码的上游返回值经过长度和类型校验后才发送给浏览器。

## 降级行为

- 浏览器拒绝定位：使用 Cloudflare 大致位置和城市名；
- 精确坐标成功但反向地理编码失败：天气仍按精确坐标显示，地名显示“当前位置”；
- 天气请求失败：保留上一份成功结果，并显示“更新暂时失败”；
- 音乐列表为空：浮层内显示既有空状态；
- 网易云播放器加载失败：保留歌曲信息和网易云外链；
- JavaScript 不可用：浮层展开，音乐列表和外链可用，天气显示服务端占位状态；
- `prefers-reduced-motion`：抽屉只做即时状态切换，不播放位移动画。

## 测试与完成标准

自动化测试覆盖：

- Hero 内存在天气音乐浮层，站点近况后不再出现独立天气音乐大区块；
- 桌面默认展开、手机默认收起；
- 展开按钮的 `aria-expanded`、`aria-controls`、44 像素触控尺寸和键盘操作；
- 用户选择通过 `localStorage` 恢复；
- 收起时内部控件不可聚焦，重新展开后恢复；
- 收起后已创建的 iframe 仍保持单实例，重新展开可继续操作；
- 精确坐标使用反向地理编码的区县和城市，不再使用 Cloudflare IP 城市名；
- 反向地理编码失败时地名为“当前位置”，天气仍使用精确坐标；
- 定位拒绝时使用 Cloudflare 城市与坐标；
- 浮层展开时首次刷新、10 分钟刷新、页面恢复可见刷新；
- 浮层收起时不继续发起天气或定位请求；
- 请求失败时保留上一份成功天气；
- 桌面与 390 像素手机布局无横向溢出；
- Hero 文字、按钮、人物主要区域和向下阅读入口仍可见；
- 前台首页通过 WCAG A/AA Axe 检查。

完成条件：天气与音乐在 Hero 内以半透明抽屉呈现并可可靠隐藏；精确定位的地名与天气坐标一致；浮层展开时天气按 10 分钟粒度自动更新；原首页内容、后台管理、音乐数据和媒体生命周期保持可用。
