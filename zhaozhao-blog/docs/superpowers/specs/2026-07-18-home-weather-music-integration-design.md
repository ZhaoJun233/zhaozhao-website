# 首页天气与音乐整合设计

## 目标

将现有 `/now/` 页面中的天气和音乐能力迁入博客首页，同时保持首页现有 Hero、站点近况、精选文章、分类、项目和个人介绍的内容与组件不变。时间、日期和问候语不迁入首页。

前台不再将“此刻”作为独立导航页面；后台也不再将音乐作为独立管理页面，而是统一归入“页面内容”中的“首页天气与音乐”区域。

## 首页信息架构

首页继续按现有顺序渲染：

1. 首页 Hero；
2. 站点近况；
3. 新增的天气与音乐模块；
4. 精选文章；
5. 分类主题；
6. 精选项目；
7. 个人介绍。

新增模块位于站点近况和精选文章之间，使用独立组件和局部样式，不修改现有首页组件的结构、文案和视觉规则。

模块提供稳定锚点 `#weather-music`，用于旧 `/now/` 地址跳转和站内定位。

## 前台组件设计

新增首页专用组合组件 `HomeWeatherMusic.astro`，负责：

- 渲染天气区域；
- 渲染音乐播放器和歌曲列表；
- 输出天气文案配置；
- 加载前端交互脚本；
- 提供模块局部的海边色彩、玻璃质感和响应式布局。

现有 `WeatherPanel.astro` 和 `MusicPlayer.astro` 继续复用。`NowClock.astro` 不在首页渲染。

桌面端采用左右双栏：天气为较紧凑的深色玻璃卡，音乐为主要内容区。手机端改为单列，顺序为天气、当前歌曲、歌曲列表。模块不得造成横向溢出，主要按钮保持至少 44 像素的可触达高度。

现有 `/now/` 页面中的样式不能直接依赖 `html[data-page="now"]`。天气与音乐所需颜色变量应下沉到新模块根节点，避免影响首页全局主题。独立页面专用的全屏高度、顶部页头补偿和时间窗景样式不迁入首页。

## 前端交互与数据流

首页服务端加载时，在现有文章、项目和页面内容之外读取启用的音乐记录，并传入首页天气与音乐组件。

天气流程保持不变：

1. 首页脚本检测天气模块；
2. 浏览器定位可用时请求访客授权；
3. 获得坐标后调用固定的 `/api/weather/`；
4. 拒绝或失败时由接口使用 Cloudflare 大致位置降级；
5. 将规范化天气结果填入天气区域。

天气接口、Open-Meteo 上游、坐标验证、超时和十分钟缓存策略不变。

音乐流程保持不变：

1. 服务端输出启用歌曲列表和网易云外链；
2. 首页初次加载不创建 iframe，也不自动播放；
3. 访客点击歌曲后更新当前歌曲信息；
4. 页面只保留一个网易云嵌入播放器实例；
5. 真实播放控制继续由网易云播放器负责。

将现有 `now-page.ts` 拆分为独立的天气初始化和音乐初始化模块，首页只加载这两部分；时间与昼夜主题逻辑不在首页执行。首页缺少时间节点时不得产生运行时错误。

## 路由与导航兼容

运行时导航和默认导航数据中移除“此刻”项。由于生产导航来自 D1 `site_settings`，需要新增迁移，安全删除指向 `/now/` 的导航记录，并同步更新本地种子数据。

`/now/` 不直接删除，而是返回永久重定向至 `/#weather-music`，以兼容历史链接、浏览器书签和搜索引擎缓存。站点地图不再将 `/now/` 作为独立内容页面列出。

## 后台信息架构

后台侧栏移除独立“音乐”导航项。`/admin/content/` 新增“首页天气与音乐”管理区域，集中维护：

- 天气标题和天气寄语；
- 音乐区标题、说明、空状态和网易云外链文案；
- 歌曲新增、编辑、启停、排序和删除；
- 歌曲封面上传、预览、更换和移除。

现有 `now_page` 设置键继续保留，避免变更数据库设置结构、备份格式和旧数据兼容逻辑。后台将该设置键显示为“首页天气与音乐”，不再使用“此刻页面”的表述。

音乐管理界面应抽成可复用后台组件并嵌入 `/admin/content/`。新增歌曲按钮必须位于音乐管理根节点内，或由脚本从文档范围查询，修复当前按钮处于脚本查询范围之外的问题。

音乐 API、D1 `music_tracks` 表、媒体引用保护、清理队列和备份结构保持不变。

`/admin/music/` 保留为兼容重定向，跳转至 `/admin/content/#home-weather-music`。

## 降级与错误处理

- 天气不可用时只显示天气局部错误，音乐和首页其他内容继续工作；
- 浏览器拒绝定位时不重复请求权限，使用既有 Cloudflare 降级路径；
- 音乐列表为空时显示后台可维护的首页空状态；
- 网易云 iframe 加载失败时保留歌曲信息和网易云外链；
- JavaScript 不可用时仍输出歌曲列表、音乐外链和天气占位结构；
- 后台保存文案或音乐失败时，各自显示局部错误，不清空用户已填写内容；
- 旧前后台路由的重定向不得形成循环。

## 数据与部署影响

不新增音乐或天气业务表，不改变现有 API 响应和备份版本。新增迁移只处理导航设置中的 `/now/` 项，必须兼容导航项不存在或已经被管理员修改的情况。

部署前需要重新生成或同步 D1 种子，并确认远程迁移不会覆盖其他自定义导航内容。

## 测试与完成标准

自动化测试按测试先行方式覆盖：

- 首页包含天气与音乐模块，现有首页地标仍存在；
- 首页不渲染时间、日期或问候语；
- 首页初次加载不创建网易云 iframe；
- 点击不同歌曲时始终只有一个 iframe；
- 定位成功、拒绝和天气失败时的降级行为；
- 桌面端和 390 像素手机端的模块顺序、可触达尺寸和无横向溢出；
- 主导航不再显示“此刻”；
- `/now/` 重定向至 `/#weather-music`；
- 后台“页面内容”能够维护 `now_page` 文案和完整音乐 CRUD；
- 后台侧栏不再显示独立“音乐”；
- `/admin/music/` 重定向至新的管理区域；
- 编辑歌曲后点击“新增歌曲”会正确清空表单；
- 现有天气接口、音乐仓库、媒体生命周期和备份测试继续通过；
- 首页及后台管理区域通过 WCAG A/AA 自动化检查。

完成条件：访客无需进入独立页面即可在首页查看天气并选择音乐，首页原有内容和顺序保持完整；管理员无需进入独立音乐页面即可在“页面内容”中维护相关文案、歌曲和封面；历史前后台地址均能安全跳转到新位置。
