# 内容维护说明

## 登录后台

访问 `/admin/`，输入 Cloudflare Secret `ADMIN_PASSWORD`。后台只有一个管理员账号；修改 `ADMIN_SESSION_SECRET` 会让现有登录立即失效。

## 日常维护

1. 在对应模块修改内容并保存。
2. 刷新前台确认结果。
3. 进行大批量编辑前，在“数据与备份”下载完整 JSON。
4. 图片通过页面内容编辑器上传到 Workers KV，再保存文章或页面内容。

分类仍被文章引用时会阻止删除；分类改名会在同一次 D1 批处理中同步文章。友链排序必须提交完整列表。留言邮箱仅在后台显示。

## 创建和编辑文章

1. 打开“文章”，点击“新建文章”。输入标题时，尚未手工改过的 Slug 会自动生成；一旦手工编辑 Slug，后续修改标题不会覆盖它。Slug 只使用小写英文、数字和连字符。
2. 填写摘要、分类、标签、发布日期和 Markdown 正文。新文章默认是草稿，需要公开时取消“保存为草稿”。
3. 在“文章封面”上传图片，并填写准确的“封面说明”。封面与说明必须同时存在；说明用于前台替代文本和读屏访问。
4. 在“Markdown 正文”把光标放到目标位置，再选择“上传并插入图片”。上传完成后会插入 `/media/uploads/.../` Markdown，并在“本文图库”显示正文、封面或图库用途。
5. 点击“保存文章”。切换文章、清空编辑器或导入文件时，后台会先等待正在进行的上传完成，并清理未保存草稿图片。

同一个 `/media/uploads/.../` 地址可被多篇文章引用。后台会把它识别为共享图片；删除其中一篇文章不会删除仍被其他文章引用的对象。

## 删除文章和图片清理

点击文章行的“删除”后，确认框会先显示：

- 仅由本文使用、将进入清理队列的图片数量；
- 仍由其他文章使用、会继续保留的共享图片数量。

确认后文章立即删除。独占图片会先进入 D1 清理队列，再从 KV 删除；KV 临时失败时任务会保留错误和尝试次数，由后续文章图片上传、旧图片回填、删除或导入操作继续重试。因此不要用刚删除图片的不可变缓存响应判断清理是否完成，应使用带随机查询参数的 URL 复查。

## Markdown 导入

“导入 Markdown”只读取文件并填充编辑器，不会直接创建文章。检查标题、Slug、摘要、分类、标签、日期、草稿状态和正文后，再点击“保存文章”。

- 未填写 `slug` 时，从英文文件名生成；导入后可在保存前修改。
- 未填写 `draft` 时默认勾选草稿。
- 导入会统计 `./images/example.png`、`../shared/cover.webp` 等相对图片路径，并提示“待上传的本地图片”。逐一用正文图片上传控件上传，然后用生成的 `/media/uploads/.../` 地址替换原相对路径，再保存。
- 站内 `/media/...`、外部 `https://...` 和 `data:` 图片不计入相对图片提示。

## 备份与恢复

后台导出的 v2 JSON 包含页面设置、分类、文章、项目、友链、留言，以及文章媒体元数据和文章—图片关系。导入前会先校验全部内容，再通过单次 D1 批处理替换数据；失败时不会保留半份内容。

v2 JSON 的 `mediaAssets` 只有 KV key、原始文件名、类型和大小等元数据，不包含图片二进制内容。恢复或迁移时必须另外备份并恢复 Workers KV 对象；只有 JSON 时，文章记录可以恢复，但缺失的 KV 图片不会随之出现。

建议：

- 每次重要发布前下载 JSON。
- 定期使用 Cloudflare D1 导出能力保存远程副本。
- KV 媒体单独保留对象备份；JSON 中的键值清单只用于核对。
- 回滚代码时同时确认迁移版本和 JSON 数据版本。

## 本地维护

```powershell
Copy-Item .dev.vars.example .dev.vars
npm run db:migrate:local
npm run dev
```

浏览器访问 `/rss.xml` 会显示可读的订阅页，阅读器仍可识别标准 RSS 2.0 内容。
