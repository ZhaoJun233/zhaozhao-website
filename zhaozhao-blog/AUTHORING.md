# 内容维护与发布

后台保存后，前台下一次请求即从 D1 读取最新内容，不需要重新构建。

## 后台模块

- **文章**：新增、编辑、删除、草稿、精选、分类、标签、封面、正文图片、共享图片识别和 Markdown 文件导入。
- **项目**：维护正文、状态、标签、仓库地址、演示地址和精选状态。
- **分类**：新增、改名、启停和删除；改名同步更新引用文章。
- **友链**：新增、编辑、启停、删除和排序。
- **留言**：审核、公开、隐藏、退回待审或删除。
- **页面内容**：维护个人资料、导航、首页、关于、友链、留言、鸣谢、索引文案和插画信息。
- **数据与备份**：导出或恢复完整 JSON。

个人资料和插画路径旁提供图片上传控件。上传成功后会得到 `/media/uploads/.../` 路径；保存当前区域后前台生效。

## 文章图片

点击“新建文章”后，标题会自动生成 Slug，直到手工修改 Slug 为止。新文章默认保存为草稿。

- **封面**：上传后填写“封面说明”，两者需同时保存。封面说明会成为前台图片替代文本。
- **正文图片**：把光标放在 Markdown 正文中，点击“上传并插入图片”；后台插入生成的 `/media/uploads/.../` 地址，并把图片加入本文图库。
- **共享引用**：在另一篇文章的 Markdown 中复用同一个媒体地址时，图片会被标记为共享。删除任一引用文章后，只要还有其他引用，KV 对象就继续保留。
- **删除预览**：删除文章前会显示独占与共享图片数量。独占图片进入清理队列；短暂 KV 错误会记录并在后续后台媒体操作中重试。

## Markdown 导入

后台“文章”可导入不超过 2 MiB 的 `.md` 或 `.markdown` 文件。导入只生成编辑器预览，确认字段并点击“保存文章”后才写入 D1。未填写 `draft` 时默认保存为草稿。

```markdown
---
title: 文章标题
description: 文章摘要
publishedAt: 2026-07-17
category: 开发
tags: Astro, Cloudflare
---

这里是正文。
```

Slug 默认取英文文件名，也可在 frontmatter 中填写 `slug`。重复 Slug 或校验失败时不会写入。导入预览会提示正文里的相对图片路径；先上传对应本地图片，再把相对路径替换成生成的 `/media/uploads/.../` 地址。

## 备份边界

后台 v2 JSON 会导出文章媒体元数据与引用关系，但不嵌入 Workers KV 图片内容。迁移、灾备和完整恢复需要同时保留 JSON/D1 数据与独立的 KV 对象备份。
